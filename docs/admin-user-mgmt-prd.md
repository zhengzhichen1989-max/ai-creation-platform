# PRD：管理员后台 - 用户管理模块

## 1. 项目信息

| 字段 | 值 |
|------|-----|
| Language | 中文 |
| 技术栈 | Fastify + sql.js + React + MUI |
| Project Name | ai-creation-platform |
| 原始需求 | 管理员后台增加用户管理模块，支持查看用户信息/积分余额、手动充值积分、密码重置，并补充完善管理功能 |

---

## 2. 产品定义

### 2.1 Product Goals

1. **运营可控**：管理员能全面掌控用户状态与积分流向，对异常情况（投诉、误扣、活动奖励等）能快速干预
2. **用户自助**：降低密码重置等高频问题的运营介入成本，让用户在无邮件服务的前提下也能自主恢复账号
3. **安全合规**：管理员操作留痕、权限边界清晰，防止越权操作和积分漏洞

### 2.2 User Stories

| # | 角色 | 故事 |
|---|------|------|
| US1 | 管理员 | 作为管理员，我想查看所有注册用户的列表及其积分余额，以便掌握平台用户概况 |
| US3 | 管理员 | 作为管理员，我想手动给指定用户充值积分（并备注原因），以便处理活动奖励、补偿等运营场景 |
| US4 | 用户 | 作为忘记密码的用户，我想通过安全问题或管理员生成临时链接来重置密码，以便重新获得账号访问权 |
| US5 | 管理员 | 作为管理员，我想禁用/启用用户账号，以便处理违规用户或临时冻结异常账号 |
| US6 | 管理员 | 作为管理员，我想查看用户的积分流水明细，以便排查积分争议或消费异常 |

### 3.3 数据模型变更

#### users 表新增字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| status | TEXT | 'active' | 用户状态：`active` / `disabled` |

#### 新增表：password_reset_tokens

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| user_id | INTEGER FK | 关联 users.id |
| token | TEXT | 一次性重置 token（UUID） |
| expires_at | TEXT | 过期时间 |
| used_at | TEXT | 使用时间，NULL 表示未使用 |
| created_at | TEXT | 创建时间 |

#### 新增表：admin_operation_logs

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| admin_id | INTEGER FK | 操作管理员 user_id |
| target_user_id | INTEGER | 目标用户 ID |
| action | TEXT | 操作类型：`credit_topup` / `password_reset` / `user_disable` / `user_enable` |
| detail | TEXT | 操作详情（JSON 字符串，如充值金额、备注等） |
| created_at | TEXT | 操作时间 |

#### credit_transactions.type 新增值

- 新增 `admin_topup`：管理员手动充值

### 3.4 密码重置流程设计

> 平台当前无邮件服务，采用**管理员生成临时链接**方案：

```
1. 用户向管理员（客服渠道）反馈忘记密码
2. 管理员在后台 → 用户详情 → 点击「重置密码」
3. 后端生成 token，写入 password_reset_tokens 表，返回一次性链接
4. 管理员复制链接，通过 IM/邮件等方式发给用户
5. 用户打开链接 → 前端展示「设置新密码」页面
6. 提交新密码 → 后端验证 token 有效性和过期时间
7. 更新 users.password_hash，标记 token 已使用
8. 重定向到登录页
```

安全要点：
- Token 为 UUID v4，不可预测
- 有效期 30 分钟，过期自动失效
- 一次性使用，使用后标记 `used_at`
- 新密码需满足最低强度要求（≥8位）

### 3.5 API 设计概要

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/users | 用户列表（分页/搜索/筛选） |
| GET | /api/admin/users/:id | 用户详情（含积分余额） |
| POST | /api/admin/users/:id/credit-topup | 手动充值积分 |
| POST | /api/admin/users/:id/reset-password | 生成密码重置链接 |
| GET | /api/admin/users/:id/transactions | 用户积分流水 |
| PUT | /api/admin/users/:id/status | 禁用/启用用户 |
| POST | /api/auth/reset-password | 用户提交新密码（公开接口，验证 token） |
